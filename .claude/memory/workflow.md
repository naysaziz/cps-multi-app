Keep the project moving fast — demo deadlines, build what's needed, nothing more.

**Why:** Building for CPS demo approval first. Optimizations come after.

**How to apply:**
- Auto-proceed: git commits, file edits, npm run scripts, prisma commands
- Ask before: `npm install` of new packages, `git push`, destructive ops, anything outside project dir
- Don't gold-plate — no audit logging, caching, or extra configurability until demo is approved
- Read only the files needed for the task — don't over-research before acting
